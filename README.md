# page-views-worker

A Cloudflare Worker that tracks page view counts for a website.  
Each time an article page loads, this Worker increments the view count by 1 and stores it in Cloudflare KV.

---

## Architecture

```
Article page loads
  → JS fetch POST { slug: "/p/post-name/" }
    → Cloudflare Worker
      → KV.get(slug) → +1 → KV.put(slug, newCount)
      → Returns { count: 42 }
  → Updates the count displayed on the page
```

---

## Initial Setup

### 1. Create a KV Namespace

Cloudflare Dashboard → Storage & Databases → KV → Create  
Enter `PAGE_VIEWS` as the name, then copy the **KV Namespace ID**.

### 2. Configure Environment Variables

Copy the `.env` template and fill in the KV Namespace ID:

```bash
cp wrangler.toml.example wrangler.toml.example  # Already exists — edit .env directly
```

Edit `.env`:

```
KV_NAMESPACE_ID=paste-your-KV-Namespace-ID-here
```

### 3. Generate wrangler.toml

```bash
chmod +x setup.sh
./setup.sh
```

This generates `wrangler.toml` (added to `.gitignore` so it won't be committed).

### 4. Install Wrangler and Log In

```bash
npm install -g wrangler
wrangler login
```

### 5. Deploy

```bash
wrangler deploy
```

After a successful deploy, the Worker URL will be shown:

```
https://page-views.YOUR_SUBDOMAIN.workers.dev
```

---

## Updating the Deployment

After modifying `src/worker.js`, redeploy with:

```bash
wrangler deploy
```
